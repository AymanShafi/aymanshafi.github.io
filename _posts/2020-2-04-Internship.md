Brief description of my experience as a Software Development Intern at [CimplyFive](https://www.cimplyfive.com/ "CimplyFive").

I was hired in the month of January 2020 along with my classmate [Anant Moudgalya](https://www.linkedin.com/in/anant-moudgalya-17a477148/ "Anant Moudgalya") for a project at CimplyFive. It involved the automation of a tediously manual set of tasks that had to be performed on a regular basis by business employees. The project we created was successful and it would complete a task that could take days and weeks in a few minutes (at an impressive accuracy!).

## Concepts I learned 

* Wrangling with PDFs! - Despite how immovable PDFs may seem, I learned that creating, reading and modifying PDFs is a very useful tool for many business organisations. So I used multiple python libraries (such as [Tika](https://github.com/chrismattmann/tika-python "Python Tika")) to parse PDFs and extract useful information from them.

* NoSQL databases - Learning how reverse indexing works was a joy! The significant performance enhancements NoSQL databases provide were greatly helpful for our use case, as it involved searching through hundreds and thousands of files.

* Designing desktop applications using Python - Although [tkinter](https://docs.python.org/3/library/tkinter.html "Python tkinter") has limited design customisation, functionality wise it covered almost everything we needed (Except *tag based input!*). Its light-weightedness helped to an good extent.

* Making use of APIs - Using the theoretical knowledge taught from college in creating a reading request and response headers was a satisfactory experience. I used *Postman* to test APIs and create a mapping between the API and functionality it provides. 

* SDLC processes like testing, logging and Daily Scrum - I wrote test cases and logged everything useful according to its log level. Daily discussions with the CTO [Roshan Hadal](https://www.linkedin.com/in/roshanhadal "Roshan Hadal") were a great indicator of whether or not I was going in the direction. He also guided us every time we were stuck with an issue for a long time. 

* **Currently**, I am developing a website for CimplyFive using the MERN stack.

In conclusion, my experience at CimplyFive, and under the guidance of Roshan has been incredibly helpful, and it gave me the confidence that *almost* every problem in Software development can be solved if I try hard enough.
